# Set up dependency injection framework

## Overview

Integrate Hilt (or Koin) for dependency injection from the start. Structuring the app with DI from the beginning avoids costly refactoring later and makes the codebase inherently testable with clear separation of concerns.

## Rationale

Retrofitting dependency injection into an existing codebase is significantly harder than starting with it. This enables clean architecture patterns for the inventory management features.
