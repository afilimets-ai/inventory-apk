# Create README.md with project overview, goals, and quick start

## Overview

The project has no README.md file — the standard entry point for any repository. Currently the only documentation is CLAUDE.md, which is an AI-assistant guidance file and not a human-facing project overview. A README.md should introduce the inventory-apk project, explain its purpose (APK inventory/management), outline the target platform (Android), and provide a quick start for developers cloning the repo.

## Rationale

README.md is the first file any developer, contributor, or stakeholder sees. Without it, there is zero discoverability of what the project does, how to run it, or how to contribute. This is the single highest-impact documentation gap for any repository, especially a greenfield project where onboarding clarity sets the trajectory for all future development.
