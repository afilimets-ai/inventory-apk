# Document project roadmap and feature specifications in version-controlled files

## Overview

The project's planning artifacts are entirely within the gitignored .auto-claude/ directory — meaning the roadmap (.auto-claude/roadmap/), specifications (.auto-claude/specs/), and insights (.auto-claude/insights/) are not version-controlled and invisible to collaborators. Furthermore, all these directories are currently empty. There is no version-controlled document describing what features the inventory-apk application will include, what the MVP scope is, or what the development milestones are.

## Rationale

Planning artifacts stored only in gitignored directories are invisible to collaborators and at risk of being lost. A version-controlled roadmap ensures all contributors understand project direction and priorities. For a greenfield project, clearly defining the MVP feature set prevents scope creep and gives developers concrete targets. The ideation_context.json confirms planned_features is an empty array, meaning no features have been formally planned yet.
