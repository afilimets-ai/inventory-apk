# Create CONTRIBUTING.md with development workflow and code standards

## Overview

There are no contributing guidelines, code style standards, or development workflow documentation. The project uses Git (with 2 commits on master branch) but has no branch strategy, commit convention, PR process, or code review guidelines documented. The .gitignore covers Auto Claude files but there is no guidance for human or AI contributors on how to participate in development.

## Rationale

As an AI-assisted development project using the Auto Claude framework, clear contributing guidelines are especially important — both human developers and AI agents need to understand the expected workflow, naming conventions, and quality gates. Establishing these norms early prevents inconsistent patterns from being set during the critical first implementations.
